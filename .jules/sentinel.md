## 2024-05-24 - Cross-Site Scripting (XSS) Vulnerability in Autocomplete Dropdown
**Vulnerability:** A Cross-Site Scripting (XSS) vulnerability was found in the autocomplete dropdown UI component in `src/ui/shared/panel-controller.js`. The `renderDropdown` method was constructing HTML strings using raw user input (`text`, `query`, and `suggestedLocator`) without escaping them and inserting the strings directly into the DOM via `innerHTML`.
**Learning:** This existed because the codebase was likely focused on string formatting (e.g., adding `<strong>` tags to highlight matching search terms) and did not include a general-purpose HTML sanitization utility to process variables before rendering.
**Prevention:** Always sanitize data fetched from external inputs, page content, or auto-generated fields before injecting it into the DOM. A new `LocatorX.utils.escapeHtml` utility function was added and should be strictly used when inserting dynamic variables into strings that are subsequently assigned to `innerHTML`.
## 2024-05-24 - Cross-Site Scripting (XSS) Vulnerability in Saved Locators Dropdown
**Vulnerability:** A Cross-Site Scripting (XSS) vulnerability was found in the saved locators dropdown UI component in `src/ui/shared/panel-controller.js`. The `savedLocators.updateDropdown()` method was constructing HTML strings using raw user input (`item.name`, `item.type`, and `item.locator`) from localStorage without escaping them and inserting the strings directly into the DOM via `innerHTML`.
**Learning:** This existed because the codebase was likely focused on string formatting and did not include a general-purpose HTML sanitization utility to process variables before rendering.
**Prevention:** Always sanitize data fetched from localStorage, external inputs, page content, or auto-generated fields before injecting it into the DOM. The `LocatorX.utils.escapeHtml` utility function was used and should be strictly used when inserting dynamic variables into strings that are subsequently assigned to `innerHTML` or HTML attributes like `title`.

## 2024-05-18 - DOM-based XSS via innerHTML in LocatorX UI
**Vulnerability:** Untrusted data from web pages (e.g., tags, attributes, text used to generate locators) was directly injected into the extension's side panel DOM via `innerHTML` without sanitization. Variables like `val`, `type`, `locator`, and `displayValue` were vulnerable.
**Learning:** The vulnerability existed because raw locator strings were placed inside template literals assigned to `.innerHTML` to construct complex table cells, rather than using `textContent` or applying escaping first. This is particularly dangerous in extensions where content scripts process potentially malicious DOM elements and pass them to privileged UI components.
**Prevention:** Always use `LocatorX.utils.escapeHtml(value)` when injecting any dynamically generated data (especially locator strings which often contain characters like `<`, `>`, `"` and `'`) into template strings that will be assigned to `innerHTML`.
